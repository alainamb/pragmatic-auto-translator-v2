How the Corpus Addition Process Will Work

Our corpus addition system will transform the traditionally manual and time-intensive process of corpus building into an efficient, AI-assisted workflow that maintains quality while dramatically reducing processing time.

Six-Step Automated Workflow

The system will guide users through a streamlined process that ensures consistency and quality across all corpus additions:

1. Document Upload & Validation

Upload documents with automatic duplicate detection and routing to appropriate domain-language folders

2. ID Generation

Automatic sequential ID creation using domain-language prefixes (e.g., gai-esp_corpus-item003)

3. Metadata Processing

AI-generated metadata with user review and editing capabilities before confirmation

4. Machine-Readable Content Generation

Schema-compliant JSON generated containing the contents of the document with options for external editing in development tools

5. File Creation & Storage

Automated saving to domain-specific processed directories with standardized naming conventions

6. Batch Vectorization

Coming Soon: Automatic vector generation when 5+ new files accumulate, creating document, section, and paragraph-level embeddings

Cross-Lingual Vector Integration

When the vectorization component is complete, the system will create sophisticated multi-level embeddings that enable cross-lingual similarity searches:

Vector Files Generated

  • gai-document-vectors.json - Whole document embeddings for discourse analysis
  • gai-section-vectors.json - Section-level vectors for contextual matching
  • gai-paragraph-vectors.json - Paragraph vectors for concept and terminology alignment

All languages will be mapped to a shared vector space using JINA Embeddings v3 for seamless cross-lingual operations.
